How it works

Auth3 is an open and trustless protocol for authorization via web3 wallets.



Server Authentication

Using private key message signing via web3 wallets, the signature verification is performed using a random nonce.


Diagram of private server authentication flow


Notice that the nonce is updated during the authentication step.
This ensures each valid signature can only be used exactly once.



Decentralized Authentication

[In Progress] It is theoretically possible to save the nonce on-chain, and perform a signature verification with every request, challenged against the public nonce.


This would remove the need for an issuer or auth token. It would however take time and cost gas.


Signature verification can be performed either on a private server (web2) or within a Dapp (web3).
